# BOJ 35185 바바와 깃발 게임
## 문제 요약
1 x N 게임판 위에서 진행되는 게임에서 게임을 시작하는 칸에 말이 있고, 게임이 종료되는 칸에 깃발이 있다.
말은  L, R이 포함된 명령어를 통해 왼쪽, 오른쪽으로 한칸씩 이동한다. 이동하고자 하는 위치가 게임판 바깥이라면,
말은 이동하지 않는다. 말이 깃발이 있는 칸과 같은 칸에 놓이는 순간 게임이 종료된다.
제시된 명령어를 모두 수행하였을 때 말이 깃발과 같은 같에 있다면 게임에서 승리하며 게임은 즉시 종료된다.
게임에서 승리할 수 있도록하는 말의 시작 위치와 깃발의 위치를 알아내면 된다.

++) N은 2보다 크거나 같고 200,000 보다 작거나 같다. 명령어의 길이 L은 1보다 크거나 같고 200,000 보다 작거나 같다.

## Naive한 접근
시작 위치로부터 가장 왼쪽 또는 가장 오른쪽으로 이동하는 경우가 마지막 명령 수행이면 된다.

## Attempts
## Attempts 1 - Accepted Solution
Naive한 접근에서 살펴 볼 수 있듯이 시작 위치로부터 가장 왼쪽 또는 가장 오른쪽으로 이동하는 경우가 마지막 수행이면 된다. 게임판의 가장 왼쪽에서 시작한다면 왼쪽으로 이동하는 명령어를 가장 많이 소모할 수 있을 것이다. 게임판의 가장 오른쪽에서 시작한다면 오른쪽으로 이동하는 명령어를 가장 많이 소모할 수 있을 것이다. 따라서 한쪽으로 가장 많이 이동하기 위해선 한쪽 끝에서 시작하는 것이 가장 이득이다. 이 두 시뮬레이션을 모두 해보아도 시간적 또는 공간적으로도 문제가 없기에 두 경우 모두 visit 리스트를 만들어 시뮬레이션 하였다.

## 구현 포인트
시뮬레이션 하는 과정은 왼쪽 끝에서 하나 오른쪽 끝에서 하나 같기에 함수로 만들어 불러오는 방식으로 코드를 최적화 하였다. "왼쪽 끝에서 시작 -> 오른쪽 끝에서 시작 -> 둘 다 승리할 수 없는 경우"로 나누는 과정에서 다음 과정으로 가야 하는 경우와 그렇지 않은 경우를 처리할 때 변수를 사용하는 것보다 exit() 함수를 활용하였다.
